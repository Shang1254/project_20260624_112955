# 班长作业审核功能使用说明

## 📋 功能概述

这是一个**智能班级管理系统**，支持班长审核学生提交的作业文件名是否符合要求。

### 核心功能

1. **教师端**
   - 发布作业时设置文件名模板（可选）
   - 查看学生提交情况
   - 审核文件名合规性

2. **班长端**
   - 审核学生提交的作业文件名
   - 通过/驳回不符合要求的提交
   - 查看审核历史和反馈

3. **学生端**
   - 查看文件名模板要求
   - 提交作业文件
   - 查看审核状态和驳回原因
   - 根据反馈重新提交

---

## 🚀 使用流程

### 第一步：数据库更新

在 **Supabase Dashboard** → **SQL Editor** 中执行 [`update_monitor_review.sql`](./update_monitor_review.sql)

脚本会自动：
- ✅ 添加 `file_name_template` 字段到作业表
- ✅ 添加审核相关字段到提交表
- ✅ 创建必要的索引
- ✅ 配置权限策略

### 第二步：设置班长角色

在数据库中为特定用户设置班长角色：

```sql
UPDATE class_members 
SET role = 'monitor' 
WHERE user_id = '你的用户 ID' 
AND class_id = '班级 ID';
```

### 第三步：发布作业

教师在发布作业时可以设置文件名模板：

**模板格式示例：**
- `学号_姓名_作业名称.docx`
- `2021001_张三_第一章练习题.pdf`
- `学号_姓名_实验报告.txt`

**占位符说明：**
- `学号` - 学生的学号
- `姓名` - 学生的姓名
- `作业名称` - 作业的具体名称

### 第四步：学生提交作业

1. 学生进入作业详情页
2. 系统会显示文件名模板要求
3. 上传文件时自动验证文件名
4. 如果文件名不合规，会提示错误信息

### 第五步：班长审核

1. 班长进入「审核文件名」页面
2. 查看所有待审核的提交
3. 点击「通过」或「驳回」
4. 驳回时需要填写原因

### 第六步：学生修改后重新提交

1. 被驳回的学生可以看到驳回原因
2. 修改文件名后重新提交
3. 等待再次审核

## 🔍 文件名验证规则

### 支持的占位符

| 占位符 | 匹配规则 | 示例 |
|--------|---------|------|
| `学号` | 数字 | 2021001, 2021002 |
| `姓名` | 中文字符 | 张三，李四 |
| `作业名称` | 任意字符 | 第一章练习题，实验报告 |

### 验证逻辑

系统会将模板转换为正则表达式进行验证：

```javascript
// 模板：学号_姓名_作业名称.docx
// 转换为：^\d+_ [\u4e00- \u9fa5]+_.+\.docx$
// 匹配：2021001_张三_第一章练习题.docx ✓
// 不匹配：作业.docx ✗
```

## 📊 审核状态说明

| 状态 | 说明 | 操作 |
|------|------|------|
| **待审核** | 学生刚提交，等待班长审核 | 通过/驳回 |
| **已通过** | 文件名合规，可以提交给老师 | 查看 |
| **已驳回** | 文件名不合规，需要重新提交 | 查看原因 |

## 💡 使用建议

### 对教师

1. **建议设置文件名模板**
   - 方便统一管理
   - 减少后续整理工作量
   
2. **明确说明命名规范**
   - 在作业描述中详细说明
   - 提供多个示例

### 对班长

1. **批量审核**
   - 按提交时间排序
   - 快速浏览文件名
   
2. **及时反馈**
   - 驳回时给出具体修改建议
   - 例如："请使用「学号_姓名」格式"

### 对学生

1. **仔细阅读要求**
   - 提交前检查文件名
   - 避免多次驳回

2. **查看驳回原因**
   - 仔细阅读反馈
   - 按要求修改后重提

## 🛠️ 技术实现

### 数据库表结构

**assignments 表新增字段：**
- `file_name_template` - 文件名模板字符串

**submissions 表新增字段：**
- `reviewer_id` - 审核人 ID
- `file_name_valid` - 文件名是否合规
- `file_name_error` - 错误信息
- `reviewed_at` - 审核时间

### API 路由

- `POST /api/assignments/[id]/submit` - 学生提交作业
- `GET /api/assignments/[id]/submissions` - 获取所有提交
- `PATCH /api/assignments/[id]/review` - 班长审核

### 前端页面

- `/assignments/[id]/submit` - 学生提交页
- `/assignments/[id]/review` - 班长审核页
- `/dashboard` - 教师审核入口

## ❓ 常见问题

### Q1: 如何取消文件名验证？

A: 发布作业时留空「文件名模板」字段即可。

### Q2: 班长和普通学生有什么区别？

A: 班长在 `class_members` 表中 `role` 字段设置为 `'monitor'`，拥有审核权限。

### Q3: 学生可以多次提交吗？

A: 当前版本只允许一次提交。如果被驳回，可以修改后重新提交。

### Q4: 如何修改驳回后的文件名？

A: 学生在提交页面可以看到驳回原因，删除旧文件，选择新命名的文件重新提交。
